## 部署选项选择

### 选项1：GitHub Pages（推荐，免费）
**适用场景**：个人项目、小型餐厅网站、希望免费托管

**部署步骤**：
1. 在 GitHub 创建新仓库，例如 `global-bites-restaurant`
2. 将所有网站文件上传到该仓库
3. 进入仓库设置 → Pages → 分支选择 `main` 或 `master` → 保存
4. 等待几分钟，GitHub 会提供一个网址，如 `https://你的用户名.github.io/global-bites-restaurant/`

**域名配置**：
1. 在仓库设置 → Pages → Custom domain → 输入您的域名（如 `www.yourdomain.com`）
2. 在您的域名注册商处，添加两条 DNS 记录：
   - 类型：CNAME，名称：www，值：你的用户名.github.io
   - 类型：A，名称：@（或空），值：185.199.108.153（GitHub Pages IP）

**优点**：
- 完全免费
- 自动 HTTPS
- 简单的版本控制
- 全球 CDN 加速

**缺点**：
- 每月带宽限制 100GB
- 不支持后端代码

### 选项2：Netlify（推荐，免费）
**适用场景**：需要自动部署、CI/CD、自定义域名功能

**部署步骤**：
1. 注册 Netlify 账户
2. 将网站文件拖放到 Netlify 控制台或连接 GitHub 仓库
3. Netlify 会自动部署并提供临时域名，如 `your-site.netlify.app`

**域名配置**：
1. 在 Netlify 控制台 → Site settings → Domain management
2. 点击 "Add custom domain"
3. 按照指引配置 DNS 记录（通常只需添加 CNAME 记录）

**优点**：
- 免费套餐包含自定义域名
- 自动 HTTPS 和 CDN
- 持续部署（每次代码更新自动重新部署）
- 表单处理功能（可用于您的联系表单）

**缺点**：
- 高级功能需付费
- 每月构建分钟数有限

### 选项3：Vercel（适合前端项目）
**部署步骤**：
1. 注册 Vercel 账户
2. 导入 GitHub/GitLab 仓库或直接上传项目
3. Vercel 会自动部署并提供 `your-site.vercel.app` 域名

**域名配置**：
1. 在 Vercel 控制台 → Settings → Domains
2. 添加您的域名并按照指示配置 DNS

**优点**：
- 专门为前端项目优化
- 自动 SSL 证书
- 全球边缘网络
- 与 Next.js 等框架深度集成

### 选项4：传统虚拟主机
**适用场景**：已有虚拟主机账户、需要更多控制权

**部署步骤**：
1. 通过 FTP 客户端（如 FileZilla）连接您的主机
2. 将所有网站文件上传到 `public_html` 或 `www` 目录
3. 确保 `index.html` 在网站根目录

**域名配置**：
1. 通常您的虚拟主机会自动处理域名关联
2. 如果使用新域名，需在主机控制面板添加域名
3. 配置 DNS 指向主机的 IP 地址或名称服务器

**优点**：
- 完全控制权
- 通常支持 PHP/MySQL 等后端技术
- 价格稳定

**缺点**：
- 需要手动管理
- 性能可能不如上述平台

### 选项5：云服务器（AWS、腾讯云、阿里云）
**适用场景**：高流量、需要定制化配置、技术能力强

**部署步骤**：
1. 购买云服务器实例
2. 配置 Web 服务器（如 Nginx 或 Apache）
3. 将网站文件上传到服务器
4. 配置服务器以服务静态文件

**优点**：
- 最高可控性
- 可扩展性强
- 适合高流量网站

**缺点**：
- 技术要求高
- 需要自行管理服务器安全
- 成本较高

## DNS 配置详细说明

无论选择哪个平台，您都需要在域名注册商处配置 DNS 记录：

### 常见 DNS 记录类型
1. **A 记录**：将域名指向 IPv4 地址
   - 名称：@（根域名）或 www
   - 值：服务器 IP 地址

2. **CNAME 记录**：将域名指向另一个域名
   - 名称：www（或其他子域名）
   - 值：托管服务提供的域名（如 `your-site.netlify.app`）

3. **NS 记录**：指定域名服务器
   - 主要用于将域名解析交给第三方 DNS 服务商

### 配置流程
1. 登录您的域名注册商控制面板（如 GoDaddy、Namecheap、阿里云、腾讯云）
2. 找到 DNS 管理或域名解析设置
3. 根据您选择的托管平台，添加相应的记录：
   - **GitHub Pages**：需添加 A 记录和 CNAME 记录
   - **Netlify/Vercel**：通常只需添加 CNAME 记录
   - **虚拟主机**：添加 A 记录指向主机 IP
4. 保存更改后，DNS 传播需要 24-48 小时

## 常见问题及解决方案

### 1. 域名访问显示空白页面
- 检查文件是否正确上传到网站根目录
- 确保 `index.html` 文件名正确（区分大小写）
- 清除浏览器缓存后重新访问

### 2. DNS 更改超过 48 小时仍未生效
- 使用在线 DNS 查询工具检查域名解析状态
- 联系域名注册商客服
- 尝试刷新本地 DNS 缓存（Windows：`ipconfig /flushdns`）

### 3. HTTPS 证书问题
- 选择支持自动 SSL 的平台（如 GitHub Pages、Netlify、Vercel）
- 确保访问时使用 `https://` 而非 `http://`
- 检查证书是否已自动颁发

### 4. 网站文件更新后不生效
- 确认新文件已上传成功
- 对于静态托管平台，可能需要清除 CDN 缓存
- 检查文件权限是否正确

### 5. 图片显示缓慢
- 考虑压缩图片大小
- 使用现代格式如 WebP
- 启用 CDN 加速
